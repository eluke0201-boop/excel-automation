# EC事業者向け 受注データ統合・集計ツール

複数のECプラットフォーム（楽天市場・Amazon・自社BASE・Yahoo!ショッピング・minne・Creema・自社Shopify など）から出力される受注CSVを、**ブラウザでドラッグ&ドロップするだけで**統合・集計・グラフ化してExcelで出力するツールです。

> 「**月末の集計作業 2時間 → 数秒**」を実現。

---

## 🎯 想定するお客さま

**副業でEC運営をしている個人事業主さま**を想定して設計しています。

> **田中 真奈美さん（32歳・東京都内在住）**
> 本業はWebデザイナー（会社員）。副業でハンドメイドアクセサリーを販売している副業EC事業者。
> 販売チャネルは「楽天市場」「Amazon」「自社BASEショップ」の3つ。

### 田中さんの「月末のあるある」

毎月末、各プラットフォームから受注CSVをダウンロードしてExcelで開く。**でも3つのCSVは列名が全然違う：**

| 楽天市場 | Amazon | 自社BASE |
|---|---|---|
| `注文日` | `purchase-date` | `order_date` |
| `商品名` | `product-name` | `item_name` |
| `個数` | `quantity` | `qty` |
| `金額` | `item-total` | `total_price` |

しかもAmazonは商品名が英語表記。「Natural Stone Pierced Earrings Rose Quartz」が「天然石ピアス（ローズクォーツ）」と同じ商品だと、人間が見て判断する必要がある。

結果：
- 列名を手作業で揃える
- Amazonの英語商品名を日本語商品名に1つずつ置き換える
- 3つのCSVをコピペで1枚に統合
- 商品別に並び替え、SUM関数で集計
- グラフを作る

**毎月の所要時間: 約2時間。**

> 「副業のはずなのに、月末の集計作業が一番しんどい…」

## 🎨 ブラウザでドラッグ&ドロップだけで完結

Streamlit ベースのWebアプリ。**Pythonの実行画面やコマンドを開く必要はありません**。クライアントは普段使いのブラウザで以下の操作だけ：

1. 📁 各ECショップから出力したCSVを **まとめてドロップ**
2. ✨ 自動でプラットフォーム判別（楽天/Amazon/...等）
3. 🚀 「集計を実行」ボタンを1クリック
4. 📥 完成したExcelを **ドラッグでデスクトップへ取り出す**

### 主な機能

| 機能 | 説明 |
|---|---|
| **🔍 自動判別** | ファイル名や列名から「これは楽天のCSVだな」と自動判定 |
| **🧠 賢い列マッピング** | `合計金額` → `金額`、`受注No` → `注文番号` などを類義語辞書で推測 |
| **➕ 新規プラットフォーム1クリック追加** | 未登録CSVをドロップすると、自動推測した内容で登録画面が開く |
| **⚙️ 設定UI内蔵** | プラットフォームの追加・編集・削除がすべてブラウザで完結（コード編集不要） |
| **🎨 色のカスタマイズ** | 各プラットフォームの色をカラーピッカーで自由に設定 |
| **📊 美しいグラフ** | Plotly でインタラクティブな円グラフ・棒グラフ・折れ線グラフ |
| **📤 ローカル完結** | データはお使いのPC内だけで処理。クラウドに送信されません |

## 🚀 クイックスタート（Webアプリ）

### セットアップ（初回のみ）

```bash
# 依存パッケージをインストール
pip install -r requirements.txt
```

### 起動

```bash
streamlit run app.py
```

ブラウザが自動で開き、http://localhost:8501 でアプリが表示されます。

### 使い方

1. **ドロップエリアに受注CSVをドラッグ&ドロップ**（複数同時可、種類は自動判別）
2. 判別結果を確認 → **「🚀 集計を実行」**をクリック
3. 結果のドーナツ/棒/折れ線グラフを確認
4. **「📊 統合レポート.xlsx」タイル**をドラッグして任意のフォルダへ取り出す

未登録のCSV形式の場合は、自動推測した内容で「🔧 設定に追加」ボタンが表示されます。クリックすると設定ダイアログが開き、確認するだけで登録完了。

## ⚙️ プラットフォームのカスタマイズ

画面右上の **歯車アイコン ⚙️** から、対応プラットフォームの追加・編集・削除がすべてブラウザでできます。

### できること

| 操作 | 説明 |
|---|---|
| **編集** | プラットフォーム名・ファイル名先頭・列名マッピング・色を変更 |
| **新規追加** | CSVをドロップして列マッピングを自動推測 → 確認して追加 |
| **削除** | 不要なプラットフォームを削除 |
| **JSON直接編集** | 上級者向けに config.json をテキストエディタで編集 |

### 列名マッピングの自動推測

新しいCSV形式をドロップすると、辞書ベースで列を推測：

| CSV列名 | 推測される統一列 |
|---|---|
| `合計金額`、`売上金額`、`代金`、`price`、`total` | → 金額 |
| `注文日時`、`受注日`、`purchase-date`、`order_date` | → 注文日 |
| `受注番号`、`注文ID`、`order-id`、`受注No` | → 注文番号 |
| `作品名`、`品名`、`product-name`、`item_name` | → 商品名 |
| `数量`、`qty`、`quantity` | → 個数 |

5項目中ほとんどを自動で埋めてくれるので、確認するだけで登録できます。

python merge_reports.py ./sample_data ./output.xlsx --config client_a_config.json

### ファイル名のルール

CLI実行時は、ファイル名の先頭でプラットフォームを識別します。設定ダイアログで定義したプレフィックスに合わせてください：

| プラットフォーム | ファイル名の例 |
|---|---|
| 楽天市場 | `rakuten_orders_202605.csv` |
| Amazon | `amazon_orders_202605.csv` |
| 自社BASE | `base_orders_202605.csv` |

Webアプリ経由なら、ファイル名が違っても**中身の列名から自動判別**します。

## 📦 出力されるExcelの構成

| シート名 | 内容 |
|---|---|
| **全注文** | 全プラットフォーム統合の完全データ。`プラットフォーム`列で由来を追跡可能。 |
| **商品別集計** | 商品ごとの注文件数・販売個数・売上合計。売上が高い順にソート、**横棒グラフ付き**。 |
| **プラットフォーム別集計** | プラットフォーム別の売上と構成比（%）、**ドーナツチャート付き**。 |
| **日別売上推移** | 注文日ごとの売上推移、**折れ線グラフ付き**。 |

ヘッダー行は紫色装飾＋固定、金額は ¥1,234 形式、偶数行に薄背景で読みやすさUP。
